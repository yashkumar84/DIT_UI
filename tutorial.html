1. ::before and ::after Pseudo-elements

Adding Content

This is a quote with decorative quotation marks added using ::before and ::after
.quote-box::before { content: '"'; position: absolute; font-size: 4rem; }

Decorative Elements

Styled Heading
.decorative-heading::before, .decorative-heading::after { content: ''; width: 50px; height: 2px; background: linear-gradient(...); }

Overlay Effects

Hover for overlay change

The overlay darkens/lightens on hover

.overlay-card::before { content: ''; background: rgba(0,0,0,0.3); position: absolute; inset: 0; }

Shape Creation

.arrow-button::after { content: ''; border-left: 15px solid #28a745; border-top: 15px solid transparent; border-bottom: 15px solid transparent; }

Loading Animation

.loading-button::before { background: linear-gradient(90deg, transparent, rgba(255,255,255,0.3), transparent); left: -100%; } :hover::before { left: 100%; }

3. Common Use Cases & Best Practices

Key Points to Remember:

::before and ::after:

  • Must have content property (even if empty '')
  • Are inline by default, use display: block if needed
  • Perfect for decorative elements, tooltips, icons
  • Don't affect document semantics or accessibility
  • Can't be selected by users

Transform & Translate:

  • transform doesn't affect document flow
  • translate(-50%, -50%) is the best way to center elements
  • Hardware accelerated (better performance than changing top/left)
  • Can combine multiple transforms: transform: translateX(10px) rotate(45deg) scale(1.2)
  • Use transform-origin to change rotation/scale point
